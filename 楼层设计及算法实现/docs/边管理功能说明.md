# 边管理功能说明

## 功能概述
在地图上点击节点后，可以选择该节点能到达的其他节点，创建边并保存到 `graph_edges` 表中。支持多选连接和解除连接。

## 使用流程

### 1. 进入边管理模式
点击工具栏中的"连接节点"按钮（🔗图标），按钮会变为紫色激活状态。

### 2. 选择起点节点
在边管理模式下，点击地图上的任意节点（房间、楼道、出入口或楼梯）作为起点。

### 3. 选择终点节点
弹出模态框，显示当前楼层所有可连接的节点列表：
- **白色背景**：未连接的节点，点击创建连接
- **浅绿色背景**：已连接的节点，显示"已连接"标签，点击解除连接

### 4. 多选连接
- 点击未连接的节点，自动创建边
- 模态框保持打开，可以继续选择其他节点
- 点击已连接的节点，确认后解除连接
- 点击"关闭"按钮退出

### 5. 自动更新
系统会自动：
- 计算两个节点之间的欧几里得距离作为权重
- 创建双向边
- 保存到数据库
- 刷新边列表和连接状态

## 功能特性

### 节点类型支持
- 🚪 房间（doors表，room_type=1）
- 🛤️ 楼道（doors表，room_type=2）
- 🚧 出入口（doors表，room_type=3）
- 🪜 楼梯（stairways表）

### 同楼层限制
只能连接同一楼层的节点，确保边的合理性。系统会自动过滤掉不同楼层的节点。

### 多选连接
支持一次选择多个节点进行连接，无需重复操作。模态框保持打开状态，直到用户主动关闭。

### 连接状态可视化
- **白色背景**：未连接的节点
- **浅绿色背景**：已连接的节点，显示"已连接"标签
- 点击已连接的节点可以解除连接

### 自动权重计算
根据节点坐标自动计算欧几里得距离：
```javascript
distance = √[(x2-x1)² + (y2-y1)²]
```

### 双向边
所有创建的边都是双向的，可以双向通行。

### 去重机制
数据库唯一约束确保不会重复创建边。

## 前端实现

### 状态管理
```javascript
const isAddingEdge = ref(false)  // 是否处于边管理模式
const selectedNodeForEdge = ref(null)  // 选中的起点节点
const showEdgeModal = ref(false)  // 是否显示边选择模态框
const availableNodes = ref([])  // 可连接的节点列表
const edges = ref([])  // 当前楼层的边列表
```

### 核心函数

#### toggleEdgeMode()
切换边管理模式，关闭其他模式。

#### onNodeClickForEdge(nodeType, nodeId, nodeName)
处理节点点击，选择起点节点并打开模态框。

#### loadAvailableNodes()
加载当前楼层所有可连接的节点：
- 过滤掉已选中的节点
- 只显示同楼层的节点（通过 mapId 过滤）

#### createEdge(targetNode)
创建边：
1. 获取两个节点的坐标
2. 计算距离作为权重
3. 调用后端API创建边
4. 刷新边列表

#### loadEdges(mapId)
加载当前楼层的所有边。

### 节点点击处理
修改了原有的节点详情点击函数，支持两种模式：
- 普通模式：显示节点详情
- 边管理模式：选择节点创建边

```javascript
function onStairwayDetailClick(stairway) {
  if (isAddingEdge.value) {
    onNodeClickForEdge(2, stairway.stairwayId, stairway.stairwayName)
  } else {
    selectedStairway.value = stairway
    showStairwayDetail.value = true
  }
}
```

## 后端API

### 创建边
```
POST /api/graph-edges
Body: {
  nodeAType: 1,  // 1-doors, 2-stairways
  nodeAId: "1",
  nodeBType: 1,
  nodeBId: "2",
  weight: 5.5
}
```

### 查询节点的邻接边
```
GET /api/graph-edges/adjacent/{nodeType}/{nodeId}
```

### 查询所有边
```
GET /api/graph-edges/list
```

### 删除边
```
DELETE /api/graph-edges/{id}
```

## 使用场景

### 场景1：连接多个节点
1. 点击"连接节点"按钮
2. 点击房间A
3. 在列表中依次点击楼道B、楼道C、楼梯D
4. 自动创建三条边：房间A ↔ 楼道B、房间A ↔ 楼道C、房间A ↔ 楼梯D
5. 点击"关闭"退出

### 场景2：解除连接
1. 点击"连接节点"按钮
2. 点击房间A
3. 看到楼道B显示为绿色（已连接）
4. 点击楼道B
5. 确认解除连接
6. 楼道B变回白色（未连接）

### 场景2：连接楼道和楼梯
1. 点击"连接节点"按钮
2. 点击楼道
3. 在列表中选择楼梯
4. 自动创建边：楼道 ↔ 楼梯

### 场景3：连接不同楼梯（跨楼层）
1. 点击"连接节点"按钮
2. 点击楼梯1
3. 在列表中选择楼梯2
4. 自动创建边：楼梯1 ↔ 楼梯2

## 注意事项

1. **边管理模式互斥**：进入边管理模式后，其他添加模式会自动关闭
2. **节点必须存在**：只能连接已存在的节点
3. **同楼层限制**：只能连接同一楼层的节点，不同楼层的节点不会显示在列表中
4. **权重自动计算**：基于坐标计算，如果坐标不存在则默认为1.0
5. **双向边**：所有边都是双向的，不需要创建反向边
6. **防重复**：数据库唯一约束防止重复创建边
7. **多选支持**：模态框保持打开，可以连续选择多个节点
8. **解除连接**：点击已连接的节点（绿色）可以解除连接

5. **去重处理**：数据库唯一约束防止重复创建
6. **连接状态检查**：实时检查节点间的连接状态
7. **多选支持**：不关闭模态框，允许连续操作
8. **用户体验**：模态框、提示信息、按钮状态、颜色区分

## 总结

边管理功能为室内导航系统提供了便捷的图构建工具。用户可以通过简单的点击操作，快速建立节点之间的连接关系。支持多选连接和可视化的连接状态，大大提高了操作效率。已连接的节点显示为绿色，点击可以解除连接，使得边的管理更加直观和灵活。


## 后端边重复检查

### 1. 边重复检查逻辑

在 `GraphEdgeService.createEdge()` 方法中添加了边重复检查逻辑：

```java
@Transactional
public GraphEdge createEdge(GraphEdge edge) {
    // 检查边是否已存在（双向检查）
    GraphEdge existingEdge = graphEdgeMapper.selectByNodes(
        edge.getNodeAType(), edge.getNodeAId(),
        edge.getNodeBType(), edge.getNodeBId()
    );
    
    if (existingEdge != null) {
        // 边已存在，抛出异常
        throw new RuntimeException("边已存在：节点 " + edge.getNodeAId() + " 和节点 " + edge.getNodeBId() + " 之间已有连接");
    }
    
    // 设置默认权重
    if (edge.getWeight() == null) {
        edge.setWeight(new BigDecimal("1.0"));
    }
    
    graphEdgeMapper.insert(edge);
    return edge;
}
```

### 2. 控制器错误处理

在 `GraphEdgeController.createEdge()` 方法中捕获异常并返回错误信息：

```java
@PostMapping
public ResponseEntity<?> createEdge(@RequestBody GraphEdge edge) {
    try {
        GraphEdge created = graphEdgeService.createEdge(edge);
        return ResponseEntity.ok(created);
    } catch (RuntimeException e) {
        // 返回错误信息给前端
        return ResponseEntity.badRequest()
                .body(Map.of("error", e.getMessage()));
    } catch (Exception e) {
        return ResponseEntity.badRequest()
                .body(Map.of("error", "创建边失败"));
    }
}
```

### 3. 前端错误处理

前端在创建边和删除边时，正确处理后端返回的错误信息：

### 4. 错误处理流程

1. **前端发送创建边请求** → POST /api/graph-edges
2. **后端检查边是否已存在** → GraphEdgeService.createEdge()
3. **如果边已存在**：
   - 抛出 RuntimeException
   - Controller 捕获异常
   - 返回 400 Bad Request，body 包含 `{ "error": "边已存在：节点 X 和节点 Y 之间已有连接" }`
4. **前端接收错误响应**：
   - 从 `error.response.data.error` 提取错误信息
   - 显示 alert 提示用户
5. **如果边不存在**：
   - 创建边
   - 返回 200 OK，body 包含创建的边对象
   - 前端刷新边列表和连接状态

### 5. 用户体验

- 当用户尝试创建已存在的边时，会看到明确的错误提示
- 错误信息包含具体的节点ID，便于定位问题
- 前端不会关闭模态框，用户可以继续选择其他节点
- 已连接的节点显示为绿色，避免用户重复创建
