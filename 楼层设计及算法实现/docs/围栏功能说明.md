# 围栏功能说明

## 功能概述

围栏功能允许在地图上绘制多边形区域，用于标识特定的房间、区域或禁区。围栏可以自定义颜色、透明度和边框样式。

## 数据库表结构

### 1. regions 表（围栏主表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| region_id | VARCHAR(50) | 围栏ID（主键） |
| region_name | VARCHAR(100) | 围栏名称 |
| map_id | VARCHAR(50) | 关联地图ID |
| description | TEXT | 描述信息 |
| color | VARCHAR(20) | 围栏颜色（默认#3B82F6） |
| fill_opacity | DECIMAL(3,2) | 填充透明度0-1（默认0.3） |
| stroke_width | DECIMAL(5,2) | 边框宽度（默认0.2） |
| is_active | BOOLEAN | 是否启用（默认true） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 2. region_coordinates 表（围栏坐标点表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| coord_id | BIGINT | 坐标点ID（主键，自增） |
| region_id | VARCHAR(50) | 围栏ID（外键） |
| point_order | INT | 顶点顺序（从1开始） |
| x_coordinate | DECIMAL(10,4) | X坐标（米） |
| y_coordinate | DECIMAL(10,4) | Y坐标（米） |
| created_at | TIMESTAMP | 创建时间 |

## 后端API接口

### 1. 获取所有围栏
```
GET /api/regions
```

**响应示例**：
```json
[
  {
    "regionId": "region_001",
    "regionName": "机房区域",
    "mapId": "floor1",
    "description": "服务器机房",
    "color": "#3B82F6",
    "fillOpacity": 0.3,
    "strokeWidth": 0.2,
    "isActive": true,
    "createdAt": "2026-04-21T10:00:00",
    "updatedAt": "2026-04-21T10:00:00"
  }
]
```

### 2. 根据ID获取围栏
```
GET /api/regions/{regionId}
```

### 3. 获取围栏的坐标点
```
GET /api/regions/{regionId}/coordinates
```

**响应示例**：
```json
[
  {
    "coordId": 1,
    "regionId": "region_001",
    "pointOrder": 1,
    "xCoordinate": 10.5,
    "yCoordinate": 20.3,
    "createdAt": "2026-04-21T10:00:00"
  },
  {
    "coordId": 2,
    "regionId": "region_001",
    "pointOrder": 2,
    "xCoordinate": 15.2,
    "yCoordinate": 20.3,
    "createdAt": "2026-04-21T10:00:00"
  }
]
```

### 4. 根据地图ID获取围栏
```
GET /api/regions/map/{mapId}
```

### 5. 创建围栏
```
POST /api/regions
```

**请求体示例**：
```json
{
  "regionId": "region_001",
  "regionName": "机房区域",
  "mapId": "floor1",
  "description": "服务器机房",
  "color": "#3B82F6",
  "fillOpacity": 0.3,
  "strokeWidth": 0.2,
  "isActive": true,
  "coordinates": [
    {
      "pointOrder": 1,
      "xCoordinate": 10.5,
      "yCoordinate": 20.3
    },
    {
      "pointOrder": 2,
      "xCoordinate": 15.2,
      "yCoordinate": 20.3
    },
    {
      "pointOrder": 3,
      "xCoordinate": 15.2,
      "yCoordinate": 25.8
    },
    {
      "pointOrder": 4,
      "xCoordinate": 10.5,
      "yCoordinate": 25.8
    }
  ]
}
```

**响应**：返回创建的围栏ID

### 6. 更新围栏
```
PUT /api/regions
```

**请求体**：与创建围栏相同，必须包含regionId

### 7. 删除围栏
```
DELETE /api/regions/{regionId}
```

## 前端实现

### 1. 数据结构

**围栏对象**：
```javascript
{
  regionId: 'region_001',
  regionName: '机房区域',
  mapId: 'floor1',
  description: '服务器机房',
  color: '#3B82F6',
  fillOpacity: 0.3,
  strokeWidth: 0.2,
  isActive: true
}
```

**坐标点对象**：
```javascript
{
  coordId: 1,
  regionId: 'region_001',
  pointOrder: 1,
  xCoordinate: 10.5,
  yCoordinate: 20.3
}
```

### 2. MapCanvas组件

**Props**：
- `regions`: 围栏列表数组
- `regionCoordinates`: 围栏坐标点对象 `{regionId: [coords]}`

**渲染逻辑**：
1. 在SVG overlay中添加围栏层（最底层，在路径和标记之下）
2. 使用`<polygon>`元素渲染每个围栏
3. 根据坐标点计算多边形的points属性
4. 在围栏中心显示围栏名称

**辅助函数**：
- `getRegionPoints(regionId)`: 获取围栏的坐标点
- `getRegionPolygonPoints(regionId)`: 将坐标点转换为SVG polygon的points字符串
- `getRegionCenter(regionId)`: 计算围栏中心点（用于显示标签）

### 3. App.vue集成

**数据加载**：
```javascript
// 加载围栏数据
async function loadRegions(mapId) {
  const response = await apiClient.get(`/regions/map/${mapId}`)
  regions.value = response.data
  
  // 加载每个围栏的坐标点
  for (const region of regions.value) {
    const coordsResponse = await apiClient.get(`/regions/${region.regionId}/coordinates`)
    regionCoordinates.value[region.regionId] = coordsResponse.data
  }
}
```

**传递给MapCanvas**：
```vue
<MapCanvas
  :regions="regions"
  :regionCoordinates="regionCoordinates"
  ...
/>
```

## 样式说明

### 围栏多边形样式
- **填充颜色**：由`color`属性指定
- **填充透明度**：由`fillOpacity`属性指定（0-1）
- **边框颜色**：与填充颜色相同
- **边框宽度**：由`strokeWidth`属性指定

### 围栏标签样式
- **位置**：围栏多边形的中心点
- **字体**：12px，加粗
- **颜色**：深灰色（#1f2937）
- **描边**：白色描边，增强可读性

### 交互效果
- **悬停**：鼠标悬停时围栏略微变亮
- **点击**：可以点击围栏查看详情（待实现）

## 使用场景

1. **房间标识**：在地图上标识不同的房间区域
2. **禁区设置**：标识禁止进入的区域
3. **功能分区**：标识不同功能的区域（如办公区、生产区等）
4. **电子围栏**：结合人员定位，实现进出围栏报警功能

## 注意事项

1. **坐标顺序**：坐标点必须按顺序排列，形成闭合多边形
2. **最少顶点**：围栏至少需要3个顶点才能形成有效的多边形
3. **坐标单位**：坐标使用米（m）作为单位，与地图的物理尺寸对应
4. **删除级联**：删除围栏时会自动删除其所有坐标点（数据库外键约束）
5. **性能优化**：大量围栏时建议按需加载，避免一次性加载所有围栏
