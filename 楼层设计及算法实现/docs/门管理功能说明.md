# 门管理功能说明

## 数据库表结构

### doors 表
存储门的位置和房间信息。

**字段说明：**
- `id`: 门的唯一标识（自增主键）
- `room_name`: 房间名称（必填）
- `map_id`: 门所在的楼层ID（必填）
- `x`: 门的X坐标，单位米（必填）
- `y`: 门的Y坐标，单位米（必填）
- `description`: 门的描述信息（可选）
- `is_active`: 是否启用，0-禁用，1-启用（默认1）
- `create_time`: 创建时间
- `update_time`: 更新时间

**索引：**
- `id`: 主键（自增）
- `map_id`: 普通索引（用于按楼层查询）
- `room_name`: 普通索引（用于按房间名搜索）

## 后端 API

### 基础路径
`/api/doors`

### API 列表

#### 1. 获取所有门
- **路径**: `GET /api/doors/list`
- **返回**: 
```json
{
  "data": [
    {
      "id": 1,
      "roomName": "会议室A",
      "mapId": "floor3",
      "x": 10.5,
      "y": 20.3,
      "description": "主入口"
    }
  ],
  "total": 1
}
```

#### 2. 根据门ID获取详情
- **路径**: `GET /api/doors/{id}`
- **参数**: id (路径参数，Long类型)
- **返回**: 门的详细信息

#### 3. 根据楼层ID获取门列表
- **路径**: `GET /api/doors/map/{mapId}`
- **参数**: mapId (路径参数)
- **返回**: 该楼层的所有门

#### 4. 根据房间名搜索门
- **路径**: `GET /api/doors/search?roomName={roomName}`
- **参数**: roomName (查询参数，支持模糊搜索)
- **返回**: 匹配的门列表

#### 5. 添加门
- **路径**: `POST /api/doors`
- **请求体**:
```json
{
  "roomName": "会议室A",
  "mapId": "floor3",
  "x": 10.5,
  "y": 20.3,
  "description": "主入口"
}
```
- **返回**: 
```json
{
  "success": true,
  "message": "添加成功"
}
```

#### 6. 更新门信息
- **路径**: `PUT /api/doors`
- **请求体**: 同添加门，需包含 id
- **返回**: 成功或失败信息

#### 7. 删除门
- **路径**: `DELETE /api/doors/{id}`
- **参数**: id (路径参数，Long类型)
- **返回**: 成功或失败信息
- **注意**: 软删除，只是将 is_active 设为 0

## 使用示例

### 添加门
```bash
curl -X POST http://localhost:8080/api/doors \
  -H "Content-Type: application/json" \
  -d '{
    "roomName": "会议室A",
    "mapId": "floor3",
    "x": 10.5,
    "y": 20.3,
    "description": "主入口"
  }'
```

### 查询楼层的所有门
```bash
curl http://localhost:8080/api/doors/map/floor3
```

### 搜索房间
```bash
curl http://localhost:8080/api/doors/search?roomName=会议室
```

## 注意事项

1. `id` 是自增主键，添加时不需要提供
2. `map_id` 应该对应 maps 表中的有效楼层ID
3. 坐标 (x, y) 使用米作为单位，与地图的物理尺寸对应
4. 删除操作是软删除，数据不会真正从数据库中移除
5. 所有查询默认只返回 `is_active = 1` 的记录
