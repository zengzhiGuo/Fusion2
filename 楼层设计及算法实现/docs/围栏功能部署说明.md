# 围栏功能部署说明

## 问题诊断

如果遇到以下错误：
```
Failed to load resource: the server responded with a status of 404 (Not Found)
/api/regions/map/floor1
```

这说明后端服务还没有加载围栏相关的API。

## 部署步骤

### 1. 执行数据库脚本

在MySQL中执行建表SQL：

```bash
# 方式1：使用mysql命令行
mysql -u root -p your_database < docs/数据库变更-创建围栏表.sql

# 方式2：在MySQL客户端中执行
source docs/数据库变更-创建围栏表.sql
```

验证表是否创建成功：
```sql
SHOW TABLES LIKE 'region%';
-- 应该看到：
-- regions
-- region_coordinates

DESC regions;
DESC region_coordinates;
```

### 2. 检查后端文件

确认以下文件已创建：

**Entity层**：
- `backend/src/main/java/com/ship/entity/RegionEntity.java`
- `backend/src/main/java/com/ship/entity/RegionCoordinateEntity.java`

**DTO层**：
- `backend/src/main/java/com/ship/dto/RegionDto.java`

**Mapper层**：
- `backend/src/main/java/com/ship/mapper/RegionMapper.java`
- `backend/src/main/java/com/ship/mapper/RegionCoordinateMapper.java`
- `backend/src/main/resources/mapper/RegionMapper.xml`
- `backend/src/main/resources/mapper/RegionCoordinateMapper.xml`

**Controller层**：
- `backend/src/main/java/com/ship/controller/RegionController.java`

### 3. 编译后端项目

```bash
cd backend
mvn clean compile
```

如果编译失败，检查：
1. 是否缺少Lombok依赖
2. 包名是否正确（应该是`com.ship`）
3. MyBatis配置是否正确

### 4. 重启后端服务

```bash
# 停止当前运行的后端服务
# 然后重新启动

cd backend
mvn spring-boot:run
```

或者在IDE中重启Spring Boot应用。

### 5. 验证API是否可用

使用curl或Postman测试API：

```bash
# 测试获取所有围栏（应该返回空数组）
curl http://localhost:8080/api/regions

# 测试获取指定地图的围栏
curl http://localhost:8080/api/regions/map/floor1
```

预期响应：
```json
[]
```

如果返回404，说明Controller没有加载，检查：
1. Controller类上是否有`@RestController`注解
2. 包扫描路径是否包含`com.ship.controller`
3. 后端服务是否正常启动

### 6. 创建测试数据

使用API创建一个测试围栏：

```bash
curl -X POST http://localhost:8080/api/regions \
  -H "Content-Type: application/json" \
  -d '{
    "regionId": "test_room_001",
    "regionName": "测试房间",
    "mapId": "floor1",
    "description": "这是一个测试围栏",
    "color": "#3B82F6",
    "fillOpacity": 0.3,
    "strokeWidth": 2,
    "isActive": true,
    "coordinates": [
      {"pointOrder": 1, "xCoordinate": 5.0, "yCoordinate": 5.0},
      {"pointOrder": 2, "xCoordinate": 10.0, "yCoordinate": 5.0},
      {"pointOrder": 3, "xCoordinate": 10.0, "yCoordinate": 10.0},
      {"pointOrder": 4, "xCoordinate": 5.0, "yCoordinate": 10.0}
    ]
  }'
```

成功后应该返回围栏ID：
```
test_room_001
```

### 7. 验证前端显示

1. 刷新前端页面
2. 选择floor1地图
3. 应该能看到蓝色的测试围栏显示在地图上

## 常见问题

### Q1: 404错误 - API不存在

**原因**：后端服务没有加载Controller

**解决**：
1. 检查Controller类的包名和注解
2. 重新编译并重启后端服务
3. 检查Spring Boot的包扫描配置

### Q2: 500错误 - 数据库表不存在

**原因**：数据库表还没有创建

**解决**：
1. 执行建表SQL脚本
2. 检查数据库连接配置
3. 验证表是否创建成功

### Q3: 前端显示空白

**原因**：
1. 后端API返回空数据（正常，因为还没有创建围栏）
2. 前端渲染逻辑有问题

**解决**：
1. 创建测试围栏数据
2. 检查浏览器控制台是否有JavaScript错误
3. 检查MapCanvas组件的props是否正确传递

### Q4: 围栏显示位置不对

**原因**：坐标转换问题

**解决**：
1. 检查地图的shipLength和shipWidth是否正确
2. 检查围栏坐标是否在合理范围内（0到shipLength/shipWidth）
3. 调整围栏坐标值

### Q5: 编译错误 - 找不到Lombok

**原因**：缺少Lombok依赖

**解决**：
在`backend/pom.xml`中添加：
```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```
