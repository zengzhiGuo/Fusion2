# 房间楼道出入口区分功能说明

## 功能概述
在门管理功能基础上，通过添加 `room_type` 字段区分房间、楼道和出入口，实现统一管理但视觉区分的效果。

## 数据库变更
- 表名：`doors`
- 新增字段：`room_type` TINYINT(1) DEFAULT 1
  - 1：房间
  - 2：楼道
  - 3：出入口

## 后端实现

### 实体类 (Door.java)
```java
private Integer roomType;  // 1-房间，2-楼道，3-出入口
```

### Mapper 接口
- 新增方法：`selectByRoomType(Integer roomType)` - 按类型查询

### Controller 接口
- `GET /api/doors/type/{roomType}` - 按类型查询房间、楼道或出入口

## 前端实现

### 按钮区分
- 🚪 添加房间按钮（蓝色悬停）
- 🛤️ 添加楼道按钮（紫色悬停）
- 🚧 添加出入口按钮（红色悬停）

### 地图标记
- 房间：蓝色方块 (#3b82f6)
- 楼道：紫色方块 (#a855f7)
- 出入口：红色方块 (#ef4444)

### 表单处理
- 点击"添加房间"按钮：`roomType` 自动设置为 1
- 点击"添加楼道"按钮：`roomType` 自动设置为 2
- 点击"添加出入口"按钮：`roomType` 自动设置为 3
- 编辑时保持原有类型

### 详情显示
- 在门详情模态框中显示类型（房间/楼道/出入口）

## 使用流程

1. 选择楼层
2. 点击"添加房间"、"添加楼道"或"添加出入口"按钮
3. 在地图上点击选择位置
4. 填写名称和描述
5. 提交保存

## 技术要点

- 房间、楼道和出入口使用同一张表 `doors` 存储
- 通过 `room_type` 字段区分类型
- 前端根据 `roomType` 动态渲染不同颜色的标记
- 支持独立的添加、编辑、删除操作
- 可按类型筛选查询

## 设计目的

为室内导航路径规划提供节点基础，房间、楼道和出入口都作为图的节点参与路径计算。出入口作为特殊节点，可用于标识进出船舶的位置。
