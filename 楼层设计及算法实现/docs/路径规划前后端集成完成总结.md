# 路径规划前后端集成完成总结

## 完成时间
2026年4月17日

## 功能概述

实现了完整的前后端路径规划系统，支持：
- ✅ 两点路径规划（起点 → 终点）
- ✅ 多点路径规划（起点 → 途经点1 → 途经点2 → ... → 终点）
- ✅ 跨楼层路径规划
- ✅ 同楼层路径规划
- ✅ 实时路径规划（毫秒级响应）

## API 接口

### 1. 健康检查
```
GET /health
```

### 2. 两点路径规划
```
POST /api/plan_path
Body: {
  "start": {"x": 22.4460, "y": 13.4433, "map_id": "floor1"},
  "end": {"x": 21.1138, "y": 12.9260, "map_id": "floor2"}
}
```

### 3. 多点路径规划
```
POST /api/plan_multi_waypoint_path
Body: {
  "start": {"x": 22.4460, "y": 13.4433, "map_id": "floor1"},
  "waypoints": [
    {"x": 22.1582, "y": 6.4752, "map_id": "floor1"}
  ],
  "end": {"x": 21.1138, "y": 12.9260, "map_id": "floor2"}
}
```

### 4. RFID路径规划
```
POST /api/plan_path_from_rfid
Body: {
  "rfid_id": "RFID_001",
  "end": {"x": 21.1138, "y": 12.9260, "map_id": "floor2"}
}
```

### 5. 热加载数据
```
POST /api/reload
```

### 6. 统计信息
```
GET /api/stats
```

## 核心特性

- 调用 `/api/reload` 接口即可

## 数据流程

### 数据准备（一次性）
```bash
# 1. 从数据库导出数据
cd path_planning
python database.py

# 生成文件：
# - database_export/maps_data.txt
# - database_export/doors_data.txt
# - database_export/stairways_data.txt
# - database_export/graph_edges_data.txt
# - database_export/rfid_devices_data.txt
```

### 服务启动
```bash
# 2. 启动路径规划服务器
python server.py

# 服务器会：
# - 加载所有txt文件到内存
# - 构建图模型（126个节点，143条边）
# - 启动HTTP服务器（端口5000）
```

### 路径规划
```bash
# 3. 前端发送HTTP请求
POST http://localhost:5000/api/plan_path

# 服务器：
# - 接收起点和终点坐标
# - 查找最近的节点
# - 执行Dijkstra算法
# - 格式化路径描述
# - 返回JSON结果（<10ms）
```

## 故障排查

### 问题1：无法连接到后端
**解决**：确认后端服务器已启动，端口5000未被占用

### 问题2：路径规划失败
**解决**：检查数据库中的边连接是否完整，重新导出数据

### 问题3：CORS跨域错误
**解决**：安装flask-cors或配置前端代理
