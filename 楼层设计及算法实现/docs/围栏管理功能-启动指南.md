# 围栏管理功能启动指南

## 功能说明

围栏管理功能已完整实现，包括：
- ✅ 电子围栏的创建、编辑、删除
- ✅ 可视化地图绘制围栏
- ✅ 围栏坐标点管理
- ✅ 围栏样式自定义（颜色、透明度、边框）
- ✅ 围栏告警功能
- ✅ 围栏白名单管理

## 启动步骤

### 1. 创建数据库表

首先执行数据库脚本创建围栏相关表：

```bash
# 在MySQL中执行
mysql -u root -p your_database < docs/数据库变更-创建围栏表.sql
```

或者手动在数据库管理工具中执行 `docs/数据库变更-创建围栏表.sql` 文件。

### 2. 启动后端服务

```bash
cd 20260207/backend
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080` 启动。

### 3. 重启前端服务

**重要**：需要重启前端开发服务器才能加载新组件。

#### 方法1：停止并重启

1. 在运行前端的终端按 `Ctrl+C` 停止当前服务
2. 重新启动：

```bash
cd 20260207/frontend
npm run dev
```

#### 方法2：直接重启（推荐）

如果找不到运行的终端，可以：

1. 查找并结束node进程：
```bash
# Windows PowerShell
Get-Process | Where-Object {$_.ProcessName -eq "node"} | Stop-Process -Force

# 或者使用任务管理器结束所有 node.exe 进程
```

2. 重新启动前端：
```bash
cd 20260207/frontend
npm run dev
```

前端服务将在 `http://localhost:5173` 启动（或其他端口，查看终端输出）。

### 4. 访问围栏管理页面

1. 打开浏览器访问：`http://localhost:5173`
2. 点击左侧导航栏的 **"系统管理"**
3. 在顶部标签页中点击 **"电子围栏"**

## 功能使用说明

### 创建围栏

1. 点击 **"新建围栏"** 按钮
2. 填写基本信息：
   - 围栏ID（必填，如：fence001）
   - 围栏名称（必填，如：机房区域）
   - 关联地图（必填，从下拉列表选择）
   - 描述信息（可选）

3. 设置样式：
   - 围栏颜色（默认蓝色）
   - 填充透明度（0-1，默认0.3）
   - 边框宽度（默认0.2）
   - 状态（启用/禁用）

4. 添加坐标点（至少3个点）：
   
   **方法1：地图点选**（推荐）
   - 在弹窗底部的地图上直接点击添加坐标点
   - 拖拽顶点可调整位置
   - 右键点击顶点可删除
   - 使用"撤销"、"清空"按钮管理坐标点

   **方法2：手动输入**
   - 点击"添加坐标点"按钮
   - 手动输入X、Y坐标值（单位：米）

5. 点击 **"创建围栏"** 保存

### 编辑围栏

1. 在围栏列表中点击 **"编辑"** 按钮
2. 修改围栏信息或坐标点
3. 点击 **"更新围栏"** 保存

### 删除围栏

1. 在围栏列表中点击 **"删除"** 按钮
2. 确认删除操作

### 查看坐标JSON

点击 **"坐标JSON"** 按钮，在浏览器控制台（F12）查看围栏的坐标数据。

## 围栏白名单管理

在 **"围栏白名单"** 标签页中，可以：
- 配置人员进入围栏的权限
- 设置允许停留时间
- 管理白名单记录

## 故障排查

### 问题1：看不到"电子围栏"标签

**原因**：前端服务未重启，新组件未加载

**解决**：
1. 停止前端服务（Ctrl+C）
2. 重新运行 `npm run dev`
3. 清除浏览器缓存（Ctrl+Shift+Delete）
4. 刷新页面（Ctrl+F5）

### 问题2：点击"新建围栏"没有反应

**原因**：可能是浏览器控制台有JavaScript错误

**解决**：
1. 按F12打开浏览器开发者工具
2. 查看Console标签页的错误信息
3. 检查Network标签页的API请求是否成功

### 问题3：保存围栏时提示错误

**原因**：可能是后端服务未启动或数据库表未创建

**解决**：
1. 确认后端服务正在运行
2. 确认数据库表已创建
3. 检查后端控制台的错误日志
4. 确认至少添加了3个坐标点

### 问题4：地图不显示

**原因**：需要先创建地图

**解决**：
1. 切换到"地图管理"标签页
2. 先上传并创建地图
3. 然后再创建围栏时选择该地图

## API接口说明

### 围栏管理接口

- `GET /api/manage/regions` - 获取所有围栏
- `GET /api/manage/regions/{regionId}` - 获取单个围栏
- `GET /api/manage/regions/{regionId}/coordinates` - 获取围栏坐标
- `POST /api/manage/regions` - 创建围栏
- `PUT /api/manage/regions` - 更新围栏
- `DELETE /api/manage/regions/{regionId}` - 删除围栏

### 围栏告警接口

- `POST /api/fence/enter` - 人员进入围栏检查
- `POST /api/fence/exit` - 人员离开围栏
- `GET /api/fence/check-timeout` - 检查超时

### 白名单接口

- `GET /api/fence/whitelist` - 获取白名单
- `POST /api/fence/whitelist` - 添加白名单
- `PUT /api/fence/whitelist` - 更新白名单
- `DELETE /api/fence/whitelist/{id}` - 删除白名单

## 技术架构

### 前端技术栈
- Vue 3 + TypeScript
- Vue Router（路由管理）
- Axios（HTTP请求）
- Lucide Vue（图标库）
- Vite（构建工具）

### 后端技术栈
- Spring Boot
- MyBatis
- MySQL

### 数据库表结构
- `regions` - 围栏主表
- `region_coordinates` - 围栏坐标点表

## 注意事项

1. **坐标单位**：所有坐标值的单位都是**米**
2. **最少顶点**：围栏至少需要**3个坐标点**才能形成封闭区域
3. **地图关联**：创建围栏前必须先创建地图
4. **浏览器兼容**：建议使用Chrome、Edge或Firefox最新版本
5. **开发模式**：当前为开发模式，生产环境需要执行 `npm run build` 构建
