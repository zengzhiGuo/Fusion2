## 诊断步骤

### 步骤1: 打开浏览器开发者工具

1. 按 `F12` 打开开发者工具
2. 切换到 `Console` 标签页
3. 清空控制台（点击🚫图标）

### 步骤2: 测试路径规划功能

1. 点击 🗺️ 按钮打开路径规划面板
2. 点击"📍 点击地图添加点"按钮
3. 点击地图上的任意位置

### 步骤4: 根据日志判断问题

#### 情况A: 没有任何日志输出
**原因**: 代码未加载或有语法错误
**解决**:
1. 检查浏览器控制台是否有红色错误信息
2. 刷新页面（Ctrl+F5 强制刷新）
3. 检查 Vite 开发服务器是否正常运行

#### 情况B: 只有 `[PathPlanningPanel] toggleAddPoint` 日志
**原因**: `watch` 没有触发或 `emit` 失败
**解决**:
1. 检查 `PathPlanningPanel.vue` 中的 `watch` 是否正确
2. 检查 `defineEmits` 是否包含 `'adding-point-change'`

#### 情况C: 没有 `[MapCanvas] onMouseDown` 日志
**原因**: 点击事件未绑定或被其他元素拦截
**解决**:
1. 检查地图是否正确渲染
2. 检查是否有其他元素覆盖在地图上
3. 检查 `@mousedown` 事件是否正确绑定

#### 情况D: `[MapCanvas] isAddingPathPoint: false`
**原因**: `isAddingPathPoint` prop 未正确传递
**解决**:
1. 检查 `App.vue` 中 `MapCanvas` 的 prop 绑定
2. 检查 `isAddingPathPoint` 变量是否正确更新

#### 情况E: `[App] pathPlanningPanelRef.value: null`
**原因**: 组件引用未正确建立
**解决**:
1. 检查 `PathPlanningPanel` 是否正确渲染
2. 检查 `ref="pathPlanningPanelRef"` 是否正确设置

#### 情况F: 有所有日志但列表不更新
**原因**: Vue 响应式失效
**解决**:
1. 检查 `points` 是否是 `ref` 对象
2. 检查是否使用 `.value` 访问
3. 检查 `v-model` 绑定是否正确

## 常见问题排查

### 问题: 按钮点击无反应

**检查项**:
1. 浏览器控制台是否有错误
2. 按钮的 `@click` 事件是否正确绑定
3. 按钮是否被 CSS 遮挡（`pointer-events: none`）
4. 按钮是否被禁用（`:disabled` 属性）

**测试方法**:
```javascript
// 在控制台执行
document.querySelector('.btn-close').click()
```

### 问题: 组件未渲染

**检查项**:
1. `v-if` 条件是否为 `true`
2. 组件是否正确导入
3. 组件是否有语法错误

**测试方法**:
```javascript
// 在控制台执行
console.log(document.querySelector('.path-planning-panel'))
```

### 问题: 事件未触发

**检查项**:
1. 事件名称是否正确（kebab-case vs camelCase）
2. 事件监听器是否正确绑定
3. 事件是否被 `stopPropagation` 阻止

**测试方法**:
在组件中添加日志：
```javascript
emit('event-name', data)
console.log('事件已触发:', 'event-name', data)
```

## 快速修复建议

### 如果所有日志都正常但列表不更新

**可能原因**: `draggable` 组件问题

**临时解决方案**: 使用普通列表替代

```vue
<!-- 替换 draggable -->
<div v-else class="points-list">
  <div 
    v-for="(element, index) in points" 
    :key="element.id"
    class="point-item">
    <span class="drag-handle">☰</span>
    <span class="point-type">
      {{ index === 0 ? '🚩 起点' : index === points.length - 1 ? '🏁 终点' : `📍 途经点${index}` }}
    </span>
    <span class="point-coords">
      {{ element.map_id }} ({{ element.x.toFixed(2) }}, {{ element.y.toFixed(2) }})
    </span>
    <button class="btn-delete-point" @click="removePoint(index)">×</button>
  </div>
</div>
```

### 如果按钮无法点击

**可能原因**: CSS `z-index` 或 `pointer-events` 问题

**临时解决方案**: 添加内联样式

```vue
<button 
  class="btn-close" 
  @click="$emit('close')"
  style="position: relative; z-index: 9999; pointer-events: auto;">
  ×
</button>
```

## 下一步操作

根据控制台日志输出，确定问题出在哪个环节：

1. **如果日志显示 `addPoint` 被调用但列表不更新**
   → 问题在 Vue 响应式或 `draggable` 组件

2. **如果日志显示 `isAddingPathPoint: false`**
   → 问题在事件传递或状态管理

3. **如果没有 `onMouseDown` 日志**
   → 问题在事件绑定或元素遮挡

4. **如果按钮完全无反应**
   → 问题在 CSS 或事件监听器
