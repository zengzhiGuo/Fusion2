### 功能特性

1. **同楼层路径可视化**：在地图上绘制蓝色折线，连接路径上的所有节点
2. **跨楼层路径处理**：自动按楼层分段显示，切换楼层时显示对应楼层的路径段
3. **路径节点标记**：
   - 起点：绿色圆圈，标注"起点"
   - 终点：红色圆圈，标注"终点"
   - 途经点：蓝色小圆圈
4. **楼层导航**：路径规划面板显示所有经过的楼层，点击可快速跳转查看
5. **路径样式**：半透明蓝色折线，圆角连接，视觉效果清晰

### 技术实现

- **后端**：通过 graph.get_node_info() 获取节点坐标，添加到API返回数据中
- **前端**：
  - 使用 SVG polyline 元素绘制路径折线
  - 使用 computed 属性计算折线点坐标字符串
  - 使用 watch 监听楼层切换，自动更新路径显示
  - 使用 filter 方法按楼层筛选路径节点

## 2026/04/21 - 修复MapCanvas组件围栏显示问题

### 问题描述
前端报错：`Cannot read properties of undefined (reading 'regions')`

### 问题原因
在MapCanvas.vue的模板中使用了`props.regions`，但在Vue 3的Composition API中，setup函数中的props需要通过toRefs解构后才能在模板中直接使用。

### 修复内容

- **功能**：修复MapCanvas组件围栏显示
- **文件**：`frontend/src/components/MapCanvas.vue`
- **描述**：
  1. 导入toRefs函数
  2. 在setup中使用toRefs解构props：`const { regions, regionCoordinates, pathSegment } = toRefs(props)`
  3. 修改模板中的`props.regions`为`regions`
  4. 修改围栏相关函数使用解构后的regionCoordinates
  5. 在return中暴露regions、regionCoordinates、pathSegment和围栏相关函数
- **时间**: 2026/04/21

### 后端404问题说明
后端API返回404是因为后端服务还没有启动或没有加载新的Controller。需要：
1. 执行数据库建表SQL
2. 重新编译后端项目
3. 重启后端服务

详见：`docs/围栏功能部署说明.md`
