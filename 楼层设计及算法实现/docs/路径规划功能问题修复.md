# 路径规划功能问题修复记录

## 问题描述

**时间**: 2026/04/17 15:45

**现象**:
1. 添加完成后前端路径规划页面中看不到任何添加的信息
2. 不能点击"清空"按钮
3. 不能点击"开始规划"按钮
4. 不能点击叉号关闭页面

## 问题原因

### 根本原因
在 `App.vue` 中访问子组件 `PathPlanningPanel` 的响应式变量 `isAddingPoint` 时，使用了错误的方式：

```vue
<!-- 错误的方式 -->
:isAddingPathPoint="pathPlanningPanelRef?.isAddingPoint || false"
```

这种方式访问的是 `ref` 对象本身，而不是它的值，导致：
- `isAddingPathPoint` 始终为 `false`
- 点击地图时无法触发添加点逻辑
- 所有按钮的事件绑定失效

### 技术细节
Vue 3 的 `ref` 对象需要通过 `.value` 访问其值，但在模板中直接访问子组件 ref 的内部 ref 变量会导致响应式失效。

## 解决方案

### 方案：使用事件通信

采用 Vue 推荐的父子组件通信方式：子组件通过 `emit` 事件通知父组件状态变化。

### 修改内容

#### 1. PathPlanningPanel.vue

**新增 watch 监听**:
```javascript
import { ref, defineEmits, defineExpose, watch } from 'vue'

// 监听添加点模式变化，通知父组件
watch(isAddingPoint, (newValue) => {
  emit('adding-point-change', newValue)
})
```

#### 2. App.vue

**新增状态变量**:
```javascript
const isAddingPathPoint = ref(false)
```

**新增事件处理方法**:
```javascript
// 处理添加点模式变化
function onAddingPointChange(isAdding) {
  isAddingPathPoint.value = isAdding
  if (isAdding) {
    // 关闭其他模式
    isAddingStairway.value = false
    isAddingDoor.value = false
    isAddingCorridor.value = false
    isAddingEntrance.value = false
    isAddingRfid.value = false
    isAddingEdge.value = false
    isViewingConnections.value = false
  }
}
```

**修改点击处理逻辑**:
```javascript
// 处理路径规划点击
function onPathPointClick(coords) {
  if (pathPlanningPanelRef.value && isAddingPathPoint.value) {
    pathPlanningPanelRef.value.addPoint({
      x: coords.x,
      y: coords.y,
      map_id: selectedMap.value.mapId
    })
  }
}
```

**修改组件绑定**:
```vue
<!-- 修改前 -->
:isAddingPathPoint="pathPlanningPanelRef?.isAddingPoint || false"

<!-- 修改后 -->
:isAddingPathPoint="isAddingPathPoint"
```

**添加事件监听**:
```vue
<PathPlanningPanel
  v-if="showPathPlanning"
  ref="pathPlanningPanelRef"
  @close="closePathPlanning"
  @path-result="onPathResult"
  @adding-point-change="onAddingPointChange"
/>
```

## 经验总结

### Vue 3 组件通信最佳实践

1. **避免直接访问子组件的内部状态**
   - ❌ 不要：`childRef.value.internalState`
   - ✅ 推荐：通过 `emit` 事件通信

2. **使用 watch 监听状态变化**
   ```javascript
   watch(state, (newValue) => {
     emit('state-change', newValue)
   })
   ```

3. **父组件维护自己的状态副本**
   ```javascript
   const localState = ref(false)
   
   function onStateChange(newValue) {
     localState.value = newValue
   }
   ```

4. **通过 props 和 events 通信**
   - Props down (父 → 子)
   - Events up (子 → 父)

### 调试技巧

1. **使用 Vue DevTools**
   - 查看组件状态
   - 监听事件触发

2. **添加 console.log**
   ```javascript
   console.log('isAddingPoint:', isAddingPoint.value)
   ```

3. **检查响应式失效**
   - 确认使用 `.value` 访问 ref
   - 确认在 `setup()` 中正确返回
