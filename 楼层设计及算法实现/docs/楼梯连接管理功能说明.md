# 楼梯连接管理功能说明

## 功能概述

楼梯连接管理功能用于配置船舶不同楼层之间楼梯的连接关系，是实现跨楼层路径规划的关键功能。

## 背景

### 问题
- 原有设计中，楼梯只知道可以连接到哪个楼层（`upper_map_id`、`lower_map_id`）
- 但不知道具体连接到那个楼层的哪个楼梯
- 导致无法建立跨楼层的路径连接

### 解决方案
- 在楼梯表中添加 `upper_stairway_id` 和 `lower_stairway_id` 字段
- 明确指定楼梯连接到目标楼层的具体楼梯ID
- 提供可视化界面方便配置连接关系

## 数据库变更

### 新增字段

```sql
ALTER TABLE stairways 
ADD COLUMN upper_stairway_id VARCHAR(50) DEFAULT NULL COMMENT '上层连接的楼梯ID',
ADD COLUMN lower_stairway_id VARCHAR(50) DEFAULT NULL COMMENT '下层连接的楼梯ID';
```

### 字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| upper_stairway_id | VARCHAR(50) | 当前楼梯上楼后连接到的楼梯ID | stairway_1776095770286 |
| lower_stairway_id | VARCHAR(50) | 当前楼梯下楼后连接到的楼梯ID | stairway_1776133869885 |

### 数据约束

1. `upper_stairway_id` 指向的楼梯必须在 `upper_map_id` 楼层
2. `lower_stairway_id` 指向的楼梯必须在 `lower_map_id` 楼层
3. 如果楼梯不能上楼/下楼，对应字段为 NULL

## 后端API接口

### 1. 获取所有楼梯列表

**接口：** `GET /api/stairway-connections/stairways`

**响应示例：**
```json
[
  {
    "id": 5,
    "stairwayId": "stairway_1776095770286",
    "stairwayName": "A甲板室外左侧楼梯",
    "mapId": "floor1",
    "upperMapId": "floor2",
    "lowerMapId": null,
    "upperStairwayId": "stairway_1776134521877",
    "lowerStairwayId": null,
    "x": 13.4524,
    "y": 0.6913
  }
]
```

### 2. 根据楼层获取楼梯

**接口：** `GET /api/stairway-connections/stairways/by-floor/{mapId}`

**参数：**
- `mapId`: 楼层ID（如 floor1, floor2）

### 3. 获取楼梯连接信息

**接口：** `GET /api/stairway-connections/stairways/{stairwayId}/connections`

**响应示例：**
```json
{
  "stairway": { /* 当前楼梯信息 */ },
  "upperOptions": [ /* 上层可选楼梯列表 */ ],
  "upperSelected": { /* 当前选中的上层楼梯 */ },
  "lowerOptions": [ /* 下层可选楼梯列表 */ ],
  "lowerSelected": { /* 当前选中的下层楼梯 */ }
}
```

### 4. 更新上层连接

**接口：** `PUT /api/stairway-connections/stairways/{stairwayId}/upper-connection`

**请求体：**
```json
{
  "upperStairwayId": "stairway_1776134521877"
}
```

### 5. 更新下层连接

**接口：** `PUT /api/stairway-connections/stairways/{stairwayId}/lower-connection`

**请求体：**
```json
{
  "lowerStairwayId": "stairway_1776133869885"
}
```

### 6. 批量更新连接

**接口：** `PUT /api/stairway-connections/stairways/batch-update`

**请求体：**
```json
[
  {
    "stairwayId": "stairway_1776095770286",
    "upperStairwayId": "stairway_1776134521877",
    "lowerStairwayId": null
  },
  {
    "stairwayId": "stairway_1776133869885",
    "upperStairwayId": null,
    "lowerStairwayId": "stairway_1776133970028"
  }
]
```

### 7. 获取未配置连接的楼梯

**接口：** `GET /api/stairway-connections/stairways/unconfigured`

**说明：** 返回所有需要配置但尚未配置连接的楼梯列表

## 前端界面

### 功能特性

1. **楼梯列表展示**
   - 显示所有楼梯的基本信息
   - 标识未配置连接的楼梯（橙色边框）
   - 支持筛选只显示未配置的楼梯

2. **连接配置**
   - 为每个楼梯选择上层连接的楼梯
   - 为每个楼梯选择下层连接的楼梯
   - 下拉列表自动过滤目标楼层的楼梯

3. **统计信息**
   - 总楼梯数量
   - 未配置连接数量
   - 已配置连接数量

4. **批量保存**
   - 支持一次性保存所有更改
   - 显示待保存的更改数量
   - 保存成功后自动刷新数据

### 使用步骤

1. **打开管理界面**
   - 访问楼梯连接管理页面

2. **查看楼梯列表**
   - 查看所有楼梯及其当前连接状态
   - 橙色边框表示该楼梯需要配置连接

3. **配置连接**
   - 对于有上层连接的楼梯，从下拉列表选择目标楼梯
   - 对于有下层连接的楼梯，从下拉列表选择目标楼梯
   - 选择后会显示绿色✓标记

4. **保存更改**
   - 点击"保存所有更改"按钮
   - 系统会批量更新所有修改的连接
   - 保存成功后显示成功消息

5. **验证配置**
   - 刷新页面查看配置是否保存成功
   - 检查未配置数量是否减少

## 配置示例

### 场景：配置A甲板室外左侧楼梯

**楼梯信息：**
- 楼梯名称：A甲板室外左侧楼梯
- 楼梯ID：stairway_1776095770286
- 所在楼层：floor1
- 可以上到：floor2
- 可以下到：无

**配置步骤：**
1. 在楼梯列表中找到"A甲板室外左侧楼梯"
2. 查看"上层连接 (floor2)"下拉列表
3. 从列表中选择floor2的某个楼梯（如"B甲板室外左侧楼梯"）
4. 点击"保存所有更改"

**结果：**
- `upper_stairway_id` 字段被更新为选中的楼梯ID
- 路径规划算法可以通过这个连接实现跨楼层路径

## 注意事项

1. **连接有效性**
   - 上层连接的楼梯必须在 `upper_map_id` 指定的楼层
   - 下层连接的楼梯必须在 `lower_map_id` 指定的楼层
   - 后端会自动验证连接的有效性

2. **双向配置**
   - 如果楼梯A连接到楼梯B，建议也配置楼梯B连接回楼梯A
   - 这样可以实现双向的路径规划

3. **数据一致性**
   - 配置完成后，建议重新导出数据到txt文件
   - 确保路径规划算法使用最新的连接数据

4. **测试验证**
   - 配置完成后，运行路径规划测试验证跨楼层路径是否正常

## 与路径规划的集成

配置完楼梯连接后，路径规划算法会：

1. **读取连接信息**
   - 从数据库或txt文件读取 `upper_stairway_id` 和 `lower_stairway_id`

2. **构建跨层边**
   - 如果楼梯A的 `upper_stairway_id` 指向楼梯B
   - 在图中创建从楼梯A到楼梯B的跨层边（权重=10米）

3. **路径计算**
   - Dijkstra算法可以通过跨层边找到跨楼层的最短路径
   - 路径描述会包含"上楼"或"下楼"的动作

## 故障排查

### 问题1：保存失败

**可能原因：**
- 网络连接问题
- 后端服务未启动
- 数据库连接失败

**解决方法：**
- 检查后端服务是否运行
- 查看浏览器控制台的错误信息
- 检查后端日志

### 问题2：下拉列表为空

**可能原因：**
- 目标楼层没有楼梯
- 数据加载失败

**解决方法：**
- 检查目标楼层是否有楼梯数据
- 刷新页面重新加载数据

### 问题3：跨楼层路径仍然不可达

**可能原因：**
- 连接配置不完整
- 缺少双向连接
- 数据未同步到路径规划模块

**解决方法：**
- 检查所有楼梯的连接是否都已配置
- 确保双向连接都已配置
- 重新导出数据并重启路径规划服务
