## 1. 人员类 `Person`

| 字段名                    | 类型                   | 说明                                                         |
| ------------------------- | ---------------------- | ------------------------------------------------------------ |
| `personId`                | `String`               | 确定人员唯一标识（ZID_number、WID_number、MID_number ）      |
| `personType`              | `String`               | 人员类型：在职人员(身份确认的人员)、未识别人员(身份未确认的人员)、陌生人(不属于船上的人员) |
| `uwbTagId`                | `String`               | 关联到的 UWB 工牌 ID，(陌生人)可为空                         |
| `rfidTagId`               | `String`               | 关联到的 RFID 工牌 ID，(陌生人)可为空                        |
| `department`              | `String`               | 所属部门，如甲板部、轮机部、船舶管理部                       |
| `job`                     | `String`               | 职务，如船长、大副、轮机员、水手                             |
| `currentPosition`         | `Position`             | 当前融合坐标，包含 `x, y, 楼层z`                             |
| `track`                   | `List<Position>`       | 最近 N 帧轨迹，缓存放到内存中  (具体存储方法设计：关键时间、节点存储) |
| `inRoomId`                | `String`               | 当前所在房间 / 舱室 ID                                       |
| `baseStatus`              | `PersonBaseStatus`     | 基本状态，受状态机约束，每个状态只能按规定切换到下一个状态。（离线、在线、定位、报警、目标丢失） |
| `positionStatu`           | `PersonPositionStatus` | 人员的位置状态，受状态机约束（每个楼层中的设备是有一定顺序的） |
| `wearingStatuses`         | `Set<WearingStatus>`   | 穿着状态，可同时存在多个异常（未带安全帽、未穿救生衣、未穿工作服） |
| `behaviorStatuses`        | `Set<BehaviorStatus>`  | 行为状态，可同时存在多个行为标签（抽烟、打电话、跨越围栏、奔跑、跌倒检测、打架、睡觉 ） |
| 可信度  `confidenceScore` | String`Double`         | 当前融合结果可信度，建议 0~1                                 |
| `confidenceDesc`          | `String`               | 对可信度的解释，比如“仅UWB定位，无遮挡，置信度高”            |
| `dataSource`              | `Set<DataSourceType>`  | 当前结果来自哪些数据源，如 UWB / RFID / CAMERA  以及这些字段的可信度。 |
| `candidatePersonId`       | `String`               | 系统推测未识别人员的候选在职人员 ID                          |
| `matchReason`             | `String`               | 推测依据，如标签关联、历史轨迹连续性、摄像头识别相似度等     |
| `confidenceScore`         | `Double`               | 推测结果可信度，建议范围 0~1                                 |
| `firstSeenTime`           | `Long`                 | 首次发现时间戳 (陌生人)                                      |
| `lastUpdateTime`          | `Long`                 | 最后更新时间戳                                               |

（3）当一个人带着别的人员的标签时，能否识别出来这个人员是谁？

两个人员识别的可信度差不多时

- 视频识别算法无法判断，就将检测到的结果推给客户端，让用户自行判断，之后将判断结果返回给算法端 用于算法的学习
- 若视频识别出未检测到的人员，推断该人员是两个在职人员的可信度都差不多（根据路径和时间判断出这两个人都差不多，穿戴和行为也差不多） 先将此信息保存，检擦这两个人员下一次出现什么位置 
  - 若其中一个人员出现的位置距离保存位置很远  就不是他
  - 若其中一个人员出现的位置距离保存位置很近  就是他
  - 若两个人员出现的位置距离一样，放弃此次推断。
